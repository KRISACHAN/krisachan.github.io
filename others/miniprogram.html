          # Kbone - 微信官方推出的前端与小程序统一框架

          前两天听到有人说微信官方出了个统一Web端跟微信小程序的框架，然后马上打开文档，发现果然是如此。

          据官方介绍：

          微信小程序的底层模型和 Web 端不同，我们想直接把 Web 端的代码挪到小程序环境内执行是不可能的。

          kbone 的诞生就是为了解决这个问题，它实现了一个适配器，在适配层里模拟出了浏览器环境，让 Web 端的代码可以不做什么改动便可运行在小程序里。

          # 开发模式

          # kbone-cli

          在推出kbone的同时，微信官方也很贴心地推出了脚手架kbone-cli。

          使用方式也跟我们其他常见的前端脚手架一样：

          首先install脚手架：

          npm install -g kbone-cli

          然后创建项目：

          kbone init my-app

          最后进入项目，按照 README.md 的指引进行开发：

          // 开发小程序端
npm run mp

// 开发 Web 端
npm run web

// 构建 Web 端
npm run build

          有一点特别要注意的是，kbone-cli是基于webpack构建的，所以要使用的话，需要熟悉webpack。

          # 模板开发

          除了使用kbone-cli之外，我们也可以选择使用同样是微信官方推出的几个项目模板：

          • Vue 项目模板
          • React 项目模板
          • kbone-ui 项目模板
          • Preact 项目模板
          • Omi 项目模板

          当然，如果你对webpack有足够的了解，你可以选择手动配置开发，一般只需要补充以下两个配置：

          • 构建到小程序代码的 webpack 配置
          • 使用 webpack 构建中使用到的特殊插件 mp-webpack-plugin 配置

          # kbone-ui

          除了开发方案，当然也是少不了可供使用的多端ui库。

          kbone-ui 便是这样一个能同时支持kbone和vue开发的多端 UI 库。

          • 即可以基于 kbone 同时开发小程序和 H5，也可以单独使用开发 H5 应用。
          • 支持以 Vue 语法来支持 H5 端和小程序端运行
          • 对齐 微信weui 样式组件

          # 例子

          以官方模板 vue-kbone 我们来看看代码打包后的结果。

          vue-kbone是使用 vue 多端开发(小程序和Web)，基于 kbone 的 element 和 render。

          当我们通过模板提供的打包方式打包之后，在项目里就会多出一个dist文件夹，里面会有两个子文件夹，一个存放小程序端的生成代码，另一个则是Web端的生产代码。结果如下：

          ├─ build
├─ dist
│  ├─ mp                     // 小程序端目标代码目录，使用微信开发者工具打开，用于生产环境
│  └─ web                    // web 端编译出的文件，用于生产环境
├─ src
└─ index.html

          非常方便。

          # 如何选择？

          业内其实已经出现了很多关于同构的解决方案了，每个方案都有自己的优劣，不存在能够完美解决所有问题的方案。kbone 也一样，它的优势在上面提到过，而它的不足也是它的实现原理带来的。kbone 是使用一定的性能损耗来换取更为全面的 Web 端特性支持。

          所以关于性能方面，如果你对小程序的性能特别苛刻，建议直接使用原生小程序开发；如果你的页面节点数量特别多（通常在 1000 节点以上），同时还要保证在节点数无限上涨时仍然有稳定的渲染性能的话，可以尝试一下业内采用静态模板转译的方案；其他情况就可以直接采用 kbone 了。
